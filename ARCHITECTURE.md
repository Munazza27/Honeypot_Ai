## Component Details

### 1. **ScamDetector**
- Analyzes message text for scam indicators
- Uses keyword matching (20+ keywords)
- Pattern detection for urgency/threats
- Returns boolean: scam detected or not

### 2. **HoneyPotAgent** (AI Agent)
- Powered by Groq's Llama 3.3 70B model
- Generates human-like responses
- Maintains conversation context
- Asks questions to extract information
- Never reveals scam detection

### 3. **IntelligenceExtractor**
- Runs regex patterns on all messages
- Extracts structured data
- Deduplicates findings
- Updates continuously during conversation

### 4. **Session Manager**
- In-memory storage (dict-based)
- Tracks conversation state
- Stores all messages
- Manages agent activation

### 5. **Background Task System**
- Sends final results asynchronously
- Doesn't block response to GUVI
- Handles network errors gracefully

## External Integrations

1. **Groq API**
   - Model: llama-3.3-70b-versatile
   - Purpose: Generate human-like responses
   - Temperature: 0.8 (natural variation)

2. **GUVI Callback API**
   - URL: https://hackathon.guvi.in/api/updateHoneyPotFinalResult
   - Method: POST
   - Triggered: After conversation ends
   - Purpose: Submit final intelligence

## Scalability

- **Stateless**: Each request is independent
- **Session Storage**: In-memory (can upgrade to Redis)
- **Fast AI**: Groq provides sub-second responses
- **Railway**: Auto-scales based on traffic

## Security

- API key authentication on all endpoints
- No storage of real user data
- Ethical engagement only
- Rate limiting via Railway
